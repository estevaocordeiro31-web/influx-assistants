## 📝 CHANGELOG - MIGRAÇÃO PARA LOGIN TRADICIONAL

### Problema Original:
- Links personalizados com conflito de sessões
- Dificuldade de manter sessões separadas
- Complexidade de gerenciamento

### Solução Implementada:

**1. Schema do Banco de Dados:**
- Adicionado campo `passwordHash` na tabela `users`
- Tipo: VARCHAR(255) para armazenar hash bcrypt

**2. Backend (`server/auth-password.ts`):**
- Helper `hashPassword()` para gerar hashes
- Helper `verifyPassword()` para validar senhas
- Helper `generateRandomPassword()` para senhas aleatórias

**3. Router (`server/routers/auth-password.ts`):**
- Endpoint `login`: Autentica com email e senha
- Endpoint `changePassword`: Permite alterar senha
- Endpoint `setInitialPassword`: Admin define senha inicial

**4. Frontend (`client/src/pages/Login.tsx`):**
- Formulário de login com email e senha
- Validação em tempo real
- Mostrar/ocultar senha
- Mensagens de erro claras
- Redirecionamento automático

**5. Senhas Definidas:**
